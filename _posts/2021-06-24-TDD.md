# TDD 
    테스트 주도 개발 , 테스트가 개발을 이끌어주는 방법  
      
<br/>

# TDD 개발방법
    테스트를 만들고 테스트를 통과 시킬 수 있는 코드를 짠다  
      
<br/>

# TDD 이 필요한 순간
* 불확실한 코드를 짤때
* 외부에 의해 변경될 여지가 있을때
<br/>

#  code coverage 
    test 를 얼마나 충족시켰는가를 나타내는 지표
    code coverage tool 을 쓰려면 규칙에 맞춰 네이밍을 해야한다
    70% 이상이면 괜찮은 코드라고 볼 수 있다고 한다
